3:00 PM-3:20 PM

Learn a new project

The day starts with the game idea, build goal, controls, and the main coding skills students will use.

3:20 PM-3:50 PM

Learn more skills

Students learn extra scripts, game rules, testing habits, and how to make their project feel more complete.

3:50 PM-4:05 PM

Review the goal

Everyone checks what they are making, what they learned, and what needs to work before coding starts.

4:05 PM-5:00 PM

Code the game

Students build the core gameplay, test their work, fix bugs, add polish, and make the game playable.

5:00 PM-5:25 PM

Creative free time

Builders add their own ideas: new levels, better art, special effects, bonus rules, or surprises.

5:25 PM-5:45 PM

Fun coding

Students can make any small coding project they want, even if it is different from the main game.

5:45 PM-6:00 PM

Play time

The last 15 minutes are for playing games, testing friends' creations, and ending the day with something fun.

Student outcomes

Everyone leaves with something they made

Build Day is designed for makers who want to turn ideas into a real playable game. The format is short enough to stay focused and detailed enough to feel meaningful.

Game idea

Pick a clear challenge, goal, or world to build around.

Code

Use scripts to create movement, scoring, rules, and feedback.

Play test

Run the game, spot bugs, and improve what feels confusing.

Customize

Add personal style during free time so each game feels unique.

For Roblox leadership

A day that celebrates young builders

Build Day shows that coding can feel creative, social, and achievable. It gives students a memorable first step into building games instead of only playing them.

Build Day happens September 1. Learn, code, play test, and create.
